# TypeScript 정리

## 1. TypeScript의 설명
TypeScript(이하 TS): 마이크로소프트에서 출시한 프로그래밍 언어로 JavaScript(이하 JS)에 Static Type 시스템을 추가한 언어

Static Type 시스템: 변수나 함수가 담을 값의 자료형을 컴파일 시점에 결정하고, 정한 규칙을 컴파일러가 검사하는 체계

**TS의 핵심**
TS는 컴파일이 끝나는 순간 결정된 타입 정보가 모두 사라지고 JS로 실행된다. 즉, TS는 JS와 비교해 별도의 언어라고 하기보다는 상위 집합 언어이기때문에 JS로 작성된 코드는 TS로도 컴파일이 유효하다.

## 2.TypeScript의 특징

### 특징
- 정적 타입 검사: 변수나 함수의 값에 타입을 명시하고, 컴파일 시점에 타입 에러를 발견하여 에러를 줄이고 안정성이 높인다.
- 객체 지향 프로그래밍: 클래스, 인터페이스, 모듈 등의 OOP 기능을 지원하여 코드의 재사용과 유지보수를 보다 효율적으로 할수있게한다.
> - 클래스: 객체를 생성하기 위해 메서드를 정의한 틀
> - 인터페이스: 객체가 가져야할 속성과 타입만을 정의하는 틀
> - 모듈: export나 import로 다른 파일에서 재사용할 수 있게 나눈 코드 단위
- IDE 지원: 코드 자동 완성, 디버깅, 문법 검사등 JS에비해 다양한 기능을 활용가능하여 생산성이 높다.
- JS의 상위 확장: JS의 기능을 사용가능하며 추가로 TS의 객체지향 패턴을 사용 가능하기때문에 기존 JS코드를 그대로 사용해도 무방하다.
- 타입 추론: 타입을 적지않아도 컴파일러가 자동으로 추론을 해준다. 대신 다른 자료형이 입력되었을때 오류를 발생한다.
- 컴파일 필요: 실행 전 JS로 변환하는 과정을 필요로한다.
- ES6+ 문법 지원: 최신 JS 표준 문법도 사용가능하다.
